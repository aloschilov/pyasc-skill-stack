# Task Focus Best Practices

## When to create todo.md

- Task has more than 5 steps
- Multi-phase kernel development (always)
- Debugging sessions with multiple hypotheses
- Any task where losing context is a risk

## Update frequency

- After every completed step
- Every 3-5 steps, re-read the full todo.md to refresh context

## Template selection

| Scenario | Template |
|----------|----------|
| Kernel development | `template_kerneldev.md` |
| Debugging | `template_debug.md` |
| Simple task | `template_simple.md` |

## Anti-patterns

- Creating a todo but never updating it
- Todo items too vague ("implement stuff")
- Not re-reading the todo after long work
- Too many items (keep to 5-10 per task)
